מערכת ההפעלה

מערכת ההפעלה היא תוכנת מחשב שתפקידה העיקרי לשמש כמתווכת בין המשתמש לחומרה. היא נותנת שירות לתוכניות ולמשתמשים, מנהלת את הרצת שאר התוכניות, ומנהלת את המשאבים (זיכרון, דיסק, מעבד וכו').

ניהול זיכרון

הקצאה דינאמית של זיכרון (דברים משתנים כל הזמן), החלטה אילו חלקים של תהליכים לטעון לזיכרון ואילו לא, שמירת מידע על כל פרוסס, וניהול זיכרון וירטואלי (תוך שימוש גם בהארד דיסק).

ניהול קבצים

קבצים הם יחידת אחסון לוגית של מידע. המערכת מנהלת הרשאות גישה, יוצרת היררכיה באמצעות תיקיות, ומטפלת בכל הפעולות — יצירה, קריאה, מחיקה וכו'.

אחריות כלפי החומרה

הגנה על החומרה באמצעות מתן גישה מבוקרת מאוד. שני מצבי ריצה — user mode ו-kernel mode — ומנגנון פסיקות (interrupt) המאפשר למערכת להשתלט על המעבד. בנוסף — ייעול מקסימלי של החומרה.

💡 למה בכלל צריך מערכת הפעלה?

אם היינו יודעים שרק תוכנית אחת רצה, או שאין בה שגיאות מראש — לא היינו צריכים מערכת הפעלה. אבל אלו לא הנחות לגיטימיות כיום, ולכן נדרשת שכבת תיווך והגנה.

ארכיטקטורת פון נוימן — איך מחשב פועל?

  • מעבד (CPU): אחראי לביצוע התוכנית — מבצע את פקודות שפת המכונה המאוכסנות בזיכרון. בכל רגע נתון רצה תוכנית אחת בלבד על המעבד.
  • זיכרון: מאחסן את הפקודות לביצוע ואת הנתונים שהתוכנית משתמשת בהם.
  • רגיסטרים: תאי אחסון נתונים שעליהם מתבצעות כל הפעולות האריתמטיות והלוגיות.

האם המעבד יבצע כל פקודה? תלוי ברמת הפריווילגיה (Rings) של הפקודה הנוכחית. כשקוד מבקש לגשת לאזור בזיכרון, החומרה מאשרת לפי סוג הפקודה ולפי ה-privilege של המבקש: לכל אזור מוגדר DPL, והגישה מותרת רק אם הפריווילגיה של המבקש קטנה מה-DPL.

פסיקה (interrupt) גורמת למעבד להפסיק את ריצתו, והשליטה עוברת למערכת ההפעלה. היא נשלחת כשרוצים לבצע קריאת מערכת, כשיש שגיאה וכו'.

רמות פריווילגיה ו-PSW

הדגמה: אם הקרנל רוצה להריץ word, ה-PL של הקרנל הוא 0 ושל word הוא 3. כיוון ש-3 > 0 — הוא יכול לבצע.

📌 חשוב לזכור

ככל שה-Ring קטן יותר — רמת ההרשאה גבוהה יותר. Ring 0 = הקרנל (הכי חזק), Ring 3 = אפליקציות משתמש.

PSW הוא רגיסטר שאחד מתפקידיו להבחין אם אנחנו ב-kernel mode או user mode. אם ביט מסוים באוגר זה דולק — התהליך שרץ חסום מפעולות מסוימות (IO, גישה לכל הזיכרון, עדכון טבלאות/רגיסטרים). אם הביט כבוי — יש גישה להכל.

System Call

איך תהליך בעל privilege גבוה (Ring 3) יכול לבצע פעולות שדורשות privilege נמוך? באמצעות קריאת מערכת: מבקשים ממערכת ההפעלה שתסייע, היא מטפלת בבקשה ומחזירה את הנתונים למבקש.

בשפת C נשתמש ב-wrappers של הספרייה הסטנדרטית. לדוגמה, הדפסה ישירה ל-stdout דרך קריאת המערכת write (במקום printf שמשתמש בבאפר):

hello_syscall.c
#include <unistd.h>   /* write, STDOUT_FILENO */
#include <string.h>   /* strlen */

int main(void) {
    const char *msg = "Hello, OS!\n";

    /* קריאת מערכת ישירה: כתיבה לערוץ הפלט הסטנדרטי (fd = 1)
       write(int fd, const void *buf, size_t count); */
    write(STDOUT_FILENO, msg, strlen(msg));

    return 0;
}

הסבר: כאן אנו פונים ישירות לקרנל. STDOUT_FILENO שווה 1, msg הוא מצביע לבאפר, ו-strlen(msg) הוא מספר הבתים לכתיבה. הקלט שמועבר לקרנל: מספר ה-fd, מצביע לטקסט, ואורך. הפלט שיודפס למסך: Hello, OS!.

לינוקס ופקודות שימושיות

לינוקס היא מערכת הפעלה בהשראת Unix — חופשית, חינמית ובגישת Open Source. הפרויקט החל ב-1991 ע"י סטודנט, ומפותח בשיתוף אלפי מתכנתים. היא רצה על כמעט כל מעבד, מאפשרת מספר משתמשים במקביל ותומכת בריבוי תהליכים.

man — דפי הסבר

פקודה להצגת תיעוד על פקודות המערכת. הדפים מחולקים לסקשנים (סה"כ 9):

סקשןתיאורדוגמה
1 — User commandsפקודות משתמש רגילותls, cd
2 — System callsרשימת קריאות המערכתopen, fork
3 — C library functionsפונקציות ספרייה סטנדרטיותprintf, scanf
terminal
man ls            # דף ההסבר של ls — אפשר לראות מה עושים דגלים כמו -a
man 2 open        # מבקש במפורש את סקשן 2 (קריאת המערכת open)
man -k "copy"     # חיפוש לפי נושא במאגר המדריכים (לא זוכרים את שם הפקודה)

ניווט וניהול קבצים

terminal
ls            # מדפיס תוכן הספרייה הנוכחית
ls -al        # כולל קבצים נסתרים + פלט מורחב עם נתונים נוספים
pwd           # מדפיס את נתיב הספרייה הנוכחית

cd ..         # מעבר לספרייה שמכילה את הנוכחית
cd -          # חזרה לספרייה הקודמת שהיינו בה
cd ~          # מעבר לספריית הבית

mkdir myDir   # יצירת ספרייה חדשה
rmdir myDir   # מחיקת ספרייה (חייבת להיות ריקה! אחרת שגיאה)

mv myfile backups/      # העברת קובץ אל הספרייה backups
cp myfile backup/       # העתקת קובץ
cp *.txt /home/op/txts  # העתקת כל קובץ עם סיומת txt לתוך הספרייה txts

cat myfile              # משרשר ומדפיס תוכן קובץ למסך
cat myfile > yourfile   # מדפיס קובץ אל תוך קובץ אחר

קימפול עם gcc

terminal
gcc myprog.c            # קובץ ההרצה שייווצר יהיה a.out
gcc myprog.c -o myprog  # קביעת שם קובץ ההרצה בעצמנו
./myprog                # הרצה

הרשאות, אבטחה וקבוצות ב-Linux

לינוקס היא מערכת Multi-user — כמה משתמשים יכולים להיות מחוברים במקביל. לכל משתמש userID ייחודי. הפקודה who מציגה מי מחובר כרגע. פרטי המשתמשים נשמרים בקובץ /etc/passwd — כל שורה מתארת משתמש אחד, מחולקת ל-7 שדות מופרדים בנקודתיים (שם משתמש, סיסמה — מסומנת ב-x ושמורה בקובץ אחר מאובטח, USERID, GROUPID וכו').

קבוצות

כל משתמש יכול להשתייך למספר קבוצות (יש לו קבוצת ברירת מחדל). הקובץ /etc/group מכיל את כל הקבוצות. למה קבוצות? במקום לתת הרשאות פר-משתמש, המערכת נותנת פר-קבוצה — וכל קבוצה מקבלת "מפתח" למשאבים מסוימים.

terminal
groups guy          # אילו קבוצות שייכות למשתמש guy
newgrp developers   # החלפת קבוצת ברירת המחדל הנוכחית

שלוש הרשאות, שלוש ישויות

3 סוגי הרשאות: read=4, write=2, execute=1. הרשאה 7 = 4+2+1 (קריאה, כתיבה וביצוע). ההרשאות מתייחסות לשלוש ישויות:

  • User — בעל הקובץ. המשתמש שיצר אותו, בעל שליטה מלאה. רק הוא (ו-root) יכול לשנות הרשאות.
  • Group — הקבוצה המשויכת לקובץ; כל חבר בה מקבל את ההרשאות שהוגדרו.
  • Other — כל היתר. כאן ההרשאות הכי מצומצמות.
terminal
# מבנה: chmod [special][user][group][other] file
chmod 777 myfile    # כל מי שנוגע יכול לקרוא, לכתוב ולהריץ (rwxrwxrwx)
chmod 644 myfile    # user: rw- , group: r-- , other: r--
⚠️ umask — מסכת ההרשאות

כשתוכנה יוצרת קובץ חדש, ה-umask מגביל אוטומטית את הרשאותיו כדי למנוע הרשאות רחבות מדי. הפקודה umask מחזירה את המסכה של התהליך שהריץ אותה.

SetUID — איך מעדכנים סיסמה?

כדי לשנות סיסמה צריך לכתוב לקובץ /etc/passwd — שנמצא בבעלות root ואין למשתמש רגיל הרשאת כתיבה אליו. אז איך הסיסמה מתעדכנת? באמצעות הרשאת SetUID (האות s בהרשאות): כל מי שיש לו הרשאת גישה על הקובץ יריץ את התוכנה לא עם ההרשאות שלו, אלא עם ההרשאות של יוצר התוכנה (כאן — root). זה מאפשר לקבל כוחות root לזמן קצר שהפקודה רצה.

terminal
# ספרה רביעית מוסיפים לפני שלוש הרגילות:
#   4 → setuid (משתמש) | 2 → setgid (קבוצה) | 6 → שניהם
chmod 4755 passwd_tool   # מפעיל את ביט ה-SetUID (s) על הקובץ

📖 תרגול 1 — מחיקה של חברה שלמה בפקודה אחת

ב-2015, מרקו מרסאלה פרסם בפורום טכני שהוא בטעות הריץ פקודת rm -rf עם הרחבת משתנה שגויה על שרתי חברת האחסון שלו — ומחק את כל הנתונים של 1,535 אתרים, כולל הגיבויים שהיו מעוגנים לאותה מערכת קבצים. העסק נהרס בפקודה אחת.

זו הסיבה שחשוב להבין איך ה-shell מרחיב משתנים, איך rm עובד ברמת ה-syscall, ולמה rm לא באמת מוחק נתונים (הוא מבצע unlink ל-inode). להכיר את מערכת ההפעלה זה להבין מה פקודה באמת עושה — לא מה שאתם חושבים שהיא עושה.

המסר: פקודה אחת שגויה יכולה למחוק חברה שלמה.
