מהו ה-shell?

תוכנית שמספקת ממשק פקודות בין משתמש יחיד למערכת ההפעלה (ה-kernel). ישנן שתי משפחות:

Sh

סינטקס שמזכיר פסקל. הגרסה הפופולרית: bash.

Csh

סינטקס שמזכיר C. הגרסה הפופולרית: tcsh.

terminal
echo $SHELL   # מציג איזה shell בשימוש, למשל /bin/bash

שלושת הערוצים הסטנדרטיים

בכל תהליך פתוחים שלושה ערוצים סטנדרטיים, כל אחד מיוצג ע"י fd (File Descriptor):

fdשםתפקיד
0stdinערוץ קלט סטנדרטי — משם התוכנית קוראת נתונים.
1stdoutערוץ פלט סטנדרטי — לשם התוכנית כותבת.
2stderrערוץ שגיאות סטנדרטי — מחובר למסך כברירת מחדל, נפרד למען הודעות שגיאה דחופות.

הפניות קלט/פלט

terminal
ls > a.txt              # כותב לתוך a.txt את שמות כל הקבצים בתיקייה
                        # אם a.txt לא קיים — ה-shell יוצר אותו

echo "Hello!" > a.txt   # הפניית פלט עם דריסה: כל תוכן קודם נמחק
echo "More" >> a.txt    # הפניית פלט עם הוספה (ללא דריסה)

sort < a.txt            # הפניית קלט: מעביר את ערכי הקובץ אל sort שתמיין אותם

# << (here-document): כמה שורות קלט עד שמגיעים ל-delimiter
cat << END
Hey!
Bye!
Guy
END

wc · head · tail · pipeline

terminal
wc a.txt        # ברירת מחדל: מספר שורות, מילים, בתים + שם הקובץ
wc -l a.txt     # רק מספר שורות (סופר \n)
wc -w a.txt     # רק מספר מילים
wc -m a.txt     # רק מספר תווים
wc -c a.txt     # רק מספר בתים

head file       # 10 השורות הראשונות (ברירת מחדל)
head -5 file    # 5 השורות הראשונות
tail file       # 10 השורות האחרונות
tail -5 file    # 5 השורות האחרונות

# Pipeline: הפלט של תוכנית אחת הופך לקלט של הבאה (אין מגבלה על מספר ה-|)
ls | wc -l      # כמה קבצים יש בתיקייה הנוכחית
who | wc -l     # מספר המשתמשים המחוברים

משתנים ומשתני סביבה

משתנים מקומיים מוכרים רק ל-shell, לא לסקריפטים/פקודות אחרות.

terminal
NEWVAR="Hey"        # אתחול משתנה (שם בד"כ באותיות גדולות, ללא רווחים סביב =)
echo $NEWVAR        # התייחסות לתוכן — בעזרת הדולר → Hey

# משתני סביבה מועברים בין תוכניות ופקודות:
export NEWVAR       # הפיכת משתנה רגיל למשתנה סביבה
env                 # הצגת כל משתני הסביבה
📌 טווח השפעה

שינוי ערך משתנה סביבה משפיע רק על הטרמינל הנוכחי והתוכניות שהוא יריץ. לשינוי ברמת המערכת כולה — יש לערוך את הקובץ /etc/environment.

סקריפט — תוכנית של מפרש (interpreter)

סקריפט מורץ ע"י מפרש, ללא קומפילציה — שורה אחר שורה. מפרש נפוץ הוא bash. יתרונות: מהירות ועיבוד מחרוזות טוב.

myscript.sh
#!/bin/bash          # Shabang — אומר למערכת איזה מפרש להריץ

echo "Hello, $1"     # $1 = הארגומנט הראשון, $0 = שם הסקריפט עצמו
echo "Args: $#"      # מספר הארגומנטים שהועברו

exit 0               # נהוג ונחמד לסיים ב-exit 0
terminal
chmod +x myscript.sh
./myscript.sh Guy        # ה-shell מכניס "Guy" ל-$1 → Hello, Guy

איך מריצים סקריפטים מכל מקום?

נהפוך את הסקריפט לפקודה שמוכרת ע"י המערכת בעזרת עדכון משתנה הסביבה PATH:

terminal
mkdir $HOME/myScripts                  # שלב 1: תיקייה לכל הסקריפטים
mv dirFiles $HOME/myScripts            # שלב 2: מעבירים לתוכה את הסקריפט
PATH=$HOME/myScripts:$PATH             # שלב 3: "חפש קודם כאן, ואז בשאר המקומות"

dirFiles                               # כעת ירוץ מכל מקום כמו פקודה רגילה
⚠️ זמני בלבד

אם נסגור את הטרמינל — ה-shell ישכח את הנתיב הזה.

פקודות וסינטקס נוסף

grep

terminal
grep [options] pattern [files]
grep -c "error" log.txt    # -c → מספר השורות שמכילות את התבנית
grep -i "warn" log.txt     # -i → התעלמות מאותיות גדולות/קטנות

ביטויים אריתמטיים

terminal
num1=8
num1=$num1+7        # תוצאה כמחרוזת! → "8+7"
num1=$[$num1+7]     # סוגריים מרובעים → חישוב אמיתי

test — אמת/שקר (הפוך מהאינטואיציה!)

💡 שימו לב

בעולם ה-shell: אמת = 0, שקר = 1. המשתנה $? מחזיק את תוצאת החישוב האחרון.

terminal
test 1=2 ; echo $?    # ללא רווחים → השמה שהצליחה → 0 ("אמת")
test 1 = 2 ; echo $?  # עם רווחים → בדיקה אמיתית → 1 ("שקר")

מערכים

terminal
arr=(x1 x2 x3)       # הגדרת מערך
arr[0]="Hey!"        # השמה לאיבר
echo ${arr[2]}       # גישה לאיבר

echo ${arr[@]}       # כל האיברים — @ : ירידת שורה בין ערכים
echo ${arr[*]}       # כל האיברים — * : בשורה אחת

בקרת זרימה

control_flow.sh
#!/bin/bash

# If — הפוך מהרגיל: אם condition מחזיר 0 → מבוצע true_commands
if condition_command
then
    echo "true_commands"
else
    echo "false_commands"
fi

# While
while [ condition ]
do
    echo "running..."
done

# For
for variable in word1 word2 word3
do
    echo "$variable"
done

# פונקציות
greet() {
    echo "Hello, $1!"
}
greet "World"        # קריאה עם ארגומנט → Hello, World!

הסבר משולב: ה-if בודק קוד יציאה (0 = הצלחה = "אמת"), הלולאות עוטפות בלוק do..done, והפונקציה greet מקבלת ארגומנטים בדיוק כמו סקריפט ($1). הקלט לדוגמה האחרונה: המחרוזת "World" → הפלט: Hello, World!.

🎁 סיפור סוף התרגול

📖 תרגול 2 — Shellshock

בספטמבר 2014 התגלה באג בן 25 שנה ב-Bash עצמו — אותו shell שלמדתם לכתוב בו סקריפטים. הבאג, Shellshock (CVE-2014-6271), ניצל את האופן שבו Bash מטפל במשתני סביבה והגדרות פונקציות. תוקף יכול היה להזריק פקודות שרירותיות דרך משתני סביבה מעוצבים במיוחד.

מכיוון ש-Bash נמצא בכל מקום — שרתי ווב (CGI), SSH, לקוחות DHCP — הבאג חשף מאות מיליוני מערכות. הוא התקיים מאז Bash 1.03 ב-1989, ולקח 25 שנה עד שמישהו שם לב.

המסר: ה-Shell הוא לא רק כלי עבודה, הוא משטח תקיפה קריטי. With great power comes great responsibility!
