סמפורים (Semaphores)

קטע קריטי וסנכרון, סמפור בינארי מול מונה, פעולות אטומיות, מימוש System V (semget/semop/semctl), והשוואה בין סמפור ל-Mutex.

Critical Section

קטע בקוד שבו התהליך ניגש למשאב משותף (משתנה גלובלי, קובץ או זיכרון משותף). אם שני תהליכים ינסו לשנות את אותו משתנה בקטע זמן זה — הנתונים ישתבשו. Semaphore הוא מנגנון שמוודא שרק תהליך אחד בכל פעם נמצא בקטע הקריטי.

  • הסמפור הוא אובייקט (משתנה) שכל התהליכים רואים — נקודת התיאום ביניהם.
  • בתוכו מונה (counter). כדי להיכנס לקטע הקריטי, תהליך מוריד את המונה ב-1. אם המונה אפס — התהליך נאלץ לחכות.
  • Atomic operations: הבדיקה וההורדה קורות בפעולה אחת בלתי ניתנת לחלוקה — אי אפשר ששני תהליכים יבדקו ויורידו בו-זמנית ויחשבו ששניהם יכולים להיכנס.

שני סוגי סמפורים

בינארי

הערך 1 או 0 בלבד. מתנהג כמו מנעול — פתוח או סגור. מאפשר רק לתהליך אחד להיכנס לקטע הקריטי בכל פעם.

Counting semaphore

הערך יכול להיות כל מספר. מאפשר למספר מוגדר של תהליכים להיכנס לקטע הקריטי בו-זמנית.

איך סמפור פועל?

  • הגדרת הקטע הקריטי — מגדירים איזה חלק בקוד מסוכן.
  • יצירת סמפור שיגן על הקוד.
  • כניסה (נעילה) — הראשון נועל ונכנס. תהליכים נוספים נכנסים לרשימת ממתינים — לא מבזבזים זמן מעבד.
  • יציאה — כשהראשון מסיים, הוא מאותת לסמפור, שמעיר אחד מהממתינים ונותן לו רשות כניסה.

בלינוקס נתמקד ב-System V Semaphores. בלינוקס לא יוצרים סמפור בודד אלא מערך (סט) של סמפורים.

1. semget — יצירה/קבלת מזהה

#include <sys/sem.h> int semget(key_t key, int nsems, int semflg);
  • key — שם הסמפור במערכת. תהליכים שונים שרוצים את אותו סמפור ישתמשו באותו מפתח. IPC_PRIVATE — "הסמפור עבורי ועבור ילדיי"; תהליכים זרים לא יגשו אליו.
  • nsems — כמה סמפורים בסט. גישה לסמפור קיים → 0. יצירת חדש → גדול מ-0 (לרוב 1).
  • semflg — דגלים/הרשאות. IPC_CREAT יוצר אם לא קיים; יחד עם IPC_EXCL — נכשל אם כבר קיים.

הצלחה → מזהה ייחודי לסמפור, כישלון → −1.

2. semop — נעילה ושחרור

int semop(int semid, struct sembuf *sops, unsigned nsops);
  • semid — מזהה הסמפור מ-semget.
  • sops — מצביע למערך מבני sembuf, כל איבר מתאר פעולה על סמפור אחד בסט.
  • nsops — כמה פעולות במערך (לרוב 1).

בתוך struct sembuf שלושה שדות:

שדהמשמעות
sem_numעל איזה סמפור בסט פועלים (0 לראשון).
sem_opסוג הפעולה: חיובי (1) = שחרור משאב (מוסיף וערך ומעיר ממתינים). שלילי (−1) = נעילה (אם הערך גדול מהמוחלט — מחסירים וממשיכים, אחרת ממתינים). 0 = ממתינים עד שהסמפור יהיה בדיוק אפס.
sem_flgדגל חשוב: SEM_UNDO — אם התהליך קורס בעודו מחזיק בסמפור, המערכת תחזיר אותו אוטומטית למצבו הקודם. מונע מצב שבו תהליכים מחכים לנצח למשהו מת.

הצלחה → 0, כישלון → −1.

💡 לסיכום

כניסה לקטע קריטי → sem_op = -1. יציאה → sem_op = 1.

3. semctl — ניהול

int semctl(int semid, int semnum, int cmd, semun arg);

אתחול ערכים, קבלת מידע ומחיקת הסמפור. semnum — מס' הסמפור בסט (בפעולות על כל הסט מתעלמים). פקודות cmd נפוצות:

cmdתיאור
IPC_STAT"תביא לי דוח מצב" — שולפת מידע על הסמפור (בעלים, הרשאות) ל-ARG.
IPC_SET"עדכן הגדרות" — שינוי הרשאות גישה של סמפור קיים.
IPC_RMIDמוחקת את הסמפור מהמערכת. חשוב: אם לא נמחק — יישאר בזיכרון גם אחרי שהתוכנית תסתיים.
SETVALקביעת הערך ההתחלתי (למשל 1, שיתפקד כמנעול).
⚠️ semun — להגדיר בעצמנו

לפי תקן לינוקס, union semun לא תמיד מוגדר בספרייה. אם לא נעתיק אותו לקוד — הקומפיילר לא יזהה את הסטרקט. לכן תמיד מעתיקים:

union semun {
    int val;
    struct semid_ds *buf;
    unsigned short *array;
};

דוגמה מלאה — מנעול בינארי

sem_lock.c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/sem.h>

union semun {
    int val;
    struct semid_ds *buf;
    unsigned short *array;
};

int main(void) {
    /* 1) יצירת סט עם סמפור אחד, פרטי לתהליך וילדיו */
    int sid = semget(IPC_PRIVATE, 1, IPC_CREAT | 0666);

    /* 2) אתחול ערך הסמפור ל-1 (מנעול פתוח) */
    union semun arg;
    arg.val = 1;
    semctl(sid, 0, SETVAL, arg);

    /* 3) הגדרת פעולות wait (P) ו-signal (V) */
    struct sembuf P = { 0, -1, SEM_UNDO };  /* נעילה  */
    struct sembuf V = { 0,  1, SEM_UNDO };  /* שחרור */

    semop(sid, &P, 1);     /* --- כניסה לקטע הקריטי --- */
    /*  ... גישה בלעדית למשאב המשותף ...  */
    semop(sid, &V, 1);     /* --- יציאה --- */

    /* 4) מחיקת הסמפור מהמערכת בסיום */
    semctl(sid, 0, IPC_RMID, arg);
    return 0;
}

הסבר משולב: semget יוצר סט עם סמפור אחד. semctl(...SETVAL) מאתחל אותו ל-1 (מנעול פתוח). מבני ה-sembuf מגדירים שתי פעולות: P עם sem_op=-1 (כניסה) ו-V עם sem_op=+1 (יציאה). דגל SEM_UNDO מבטיח שאם נקרוס בתוך הקטע הקריטי — המערכת תשחרר את הסמפור אוטומטית. בסיום IPC_RMID מנקה את הסמפור מהמערכת.

📖 תרגול 6 — Mars Pathfinder

ביולי 1997, הרובר Mars Pathfinder של NASA התחיל לחוות איפוסי מערכת מסתוריים ימים אחרי הנחיתה. הסיבה? היפוך עדיפויות (priority inversion) — בעיית סנכרון. משימה בעדיפות נמוכה החזיקה mutex על אפיק משותף, משימה בעדיפות גבוהה נחסמה עליו, ומשימת תקשורת בעדיפות בינונית עקפה את הנמוכה ומנעה ממנה לשחרר את ה-mutex. טיימר ה-watchdog הבחין שהמשימה הגבוהה לא רצה ואילץ איפוס מלא.

התיקון? הפעלת priority inheritance על ה-mutex — דגל boolean ב-VxWorks, שהועלה למאדים כתוכנית C קצרה דרך interpreter שהותקן על החללית.

המסר: ניהול משאבים נכון הוא ההבדל בין הצלחה על מאדים לקריסה. זמן הוא המשאב היקר ביותר — וודאו תמיד שסדרי העדיפויות שלכם נכונים!
